价值转化

从“人工维护”转向“脚本同步 + 周期执行 + 数据可视化”，提高交付效率和数据可靠性。

优化前

  • CMDB 关联关系需要人工维护，更新滞后。
  • 配置发现脚本存在唯一性判断问题，部分实例需要手动录入。
  • 第三方系统字段缺失时，拓扑关系难以直接定位原因。

优化后

  • 听云数据可按周同步至 CMDB，支撑关联关系自动化维护。
  • 中间件、数据库等 8 类资源覆盖自动发现，涉及数百条实例。
  • 大屏数据源支撑关键指标展示，帮助项目直观呈现运维现状。

关键案例：鲸眼拓扑关联关系缺失排查

20+ 关键系统关联关系逐项梳理，定位 CMDB、听云接口和实例匹配问题。
  • 通过 Excel 梳理 20 多个关键系统的关联关系缺失情况，建立排查清单。
  • 判断问题来源：同步脚本逻辑、CMDB 数据准确性、实例业务归属、听云接口字段完整性。
  • 定位中间件接口返回类型不完整、应用实例缺少 IP、数据库实例虚 IP 与真实 IP 不一致等问题。
  • 提出处理方案：通过 APM 实例 IP 建立中间件与应用关系，推动听云侧补充数据库真实主机 IP 字段。
1. 梳理现状 按关键系统统计缺失关系，形成问题清单。
2. 定位来源 区分脚本、CMDB、接口、实例匹配等原因。
3. 设计方案 按不同数据问题给出对应处理路径。
4. 推动闭环 与相关人员沟通字段补充和后续同步逻辑。
